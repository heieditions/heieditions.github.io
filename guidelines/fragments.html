<gap>

Das Element <gap> weist in einer Transkription auf eine Stelle mit einer Textauslassung in der Vorlage in. Dies kann editorische Gründe haben, aber auch im Zustand des Originaltextes begründet liegen, zum Beispiel bei Materialverlust oder Unleserlichkeit. Das Element <gap> markiert diesen Befund und kann zudem ergänzt werden, um eine beispielsweise aus anderer Überlieferung bekannte Länge des fehlenden Begriffs auszuzeichnen. Solch eine Nutzung von <gap> kann beispielsweise so aussehen: 


               Hier ist et<gap unit="character" atLeast="2" atMost="3"/> lückenhaft.

Der Grund für die Lücke oder Textauslassung kann, soweit bekannt, vom Editor genauer benannt werden. Dies geschieht über das @rendition-Attribut direkt am <gap>-Element. heiEDITIONS stellt hierzu eine Vielzahl von Optionen zur Verfügung, um eine möglichst detailgetreue Befundbeschreibung zu ermöglichen und die digitale Edition somit möglichst nah am materiellen Textzeugen zu gestalten. Die möglichen Bezeichnungen von visuellen Gegenständen, die einer Textauslassung entsprechen, umfassen:

  • hc:Blurred: verwischt
  • hc:Burnt: verbrannt
  • hc:CutOff: abgeschnitten
  • hc:Erased: radiert (Beachten Sie: hierbei handelt es sich um eine beabsichtigte Textentfernung! Sollte der Text unbeabsichtigt entfernt worden sein, nutzen Sie bitte andere Attributwerte wie beispielsweise hc:RubbedOff.)
  • hc:Faded: verblasst
  • hc:HiddenInFold: im Falz verdeckt
  • hc:Illegible: unleserlich
  • hc:ImpairedByChemicalReagent: durch chemisches Reagens beschädigt
  • hc:Lost: verloren
  • hc:Mould: verschimmelt
  • hc:Overpainted: übermalt
  • hc:PastedOver: überklebt
  • hc:PeeledOff: abgeblättert
  • hc:Perforated: durchlöchert
  • hc:PhysicallyAffected: materiell beeinträchtigt. Diese Bezeichnung wirkt als Sammelbegriff und sollte im Idealfall nicht alleine stehen.
  • hc:Polluted: verschmutzt durch überwiegend trockenen Schmutz
  • hc:RubbedOff: abgerieben (Beachten Sie: hierbei handelt es sich um eine unbeabsichtigte Tilgung! Sollte die Textentfernung beabsichtigt gewesen sein, nutzen Sie bitte andere Attributwerte wie beispielsweise hc:Erased.)
  • hc:Stained: verschmutzt beziehungsweise befleckt mit einer Flüssigkeit
  • hc:TornOff: abgerissen

Kann der Umfang der Textauslassung erahnt werden (beispielsweise weil die grobe Form von Buchstaben trotz Verwischen noch erkannt werden kann), so besteht die Möglichkeit, Umfang und Art der Lücke am <gap>-Element anzugeben. Die Einheit der Textauslassung kann am Attribut @unit angegeben werden. Häufig wird es sich hierbei um Buchstaben handeln, was mit character angegeben wird. Zur Angabe des Länge der Textauslassung verwendet heiEDITIONS das Attribut @quantity, an welchem der genaue Umfang mittels eines Zahlenwerts angegeben werden kann: 


               Hier ist e<gap unit="character" quantity="3"/>s lückenhaft.

Ist der Umfang unbekannt und soll dieser Befund festgehalten werden, so kann dies mittels der Angabe @extent mit dem Wert "unknown" geschehen. 


               Hier ist <gap unit="character" extent="unknown"/> lückenhaft.

Fehlt ein Teil eines Begriffs durch Materialverlust oder -verschmutzung, kann aber aus dem Kontext beziehungsweise dank der erhaltenen Buchstaben mit großer Sicherheit erkannt werden, so kann er editorisch vervollständigt werden. Dies kann mithilfe des choice-Konstruktes geschehen (s. Kodierungsmethoden: Verknüpfung von Abkürzung und Auflösung. Das choice-Element).

<space>

Im Gegensatz zu <gap> bezeichnet <space> einen leeren Bereich im Textzeugen, der laut editorischem Befund im Original aus einem bestimmten Grund freigelassen wurde. Dies kann sich beispielsweise auf Freiräume beziehen, in die später Initialen eingetragen werden sollten, wobei dies aus bekannten oder unbekannten Gründen nicht durchgeführt wurde.

Um die Art des freigelassenen Raums im Original anzugeben, bietet heiEDITIONS mehrere Werte, die im Element <space> am Attribut @rendition stehen:

  • hc:BlankAreaLeftForIllustration: Freiraum für Illustration
  • hc:BlankAreaLeftForInitial: Freiraum für Initiale
  • hc:BlankAreaLeftForText: Freiraum für Text

Abb. 1: Beispiel aus dem Cod. Pal. germ. 108 für einen Freiraum, der dazu gedacht war, später mit einer Initiale gefüllt zu werden (https://doi.org/10.11588/diglit.33#0009).
